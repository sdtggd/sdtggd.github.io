一些最近写比赛项目的心得 

从24年10月份开始，我就开始在邬老师的指导下写物联网大赛的比赛项目，主要就是将前端、后端、数据库给串通起来，前端用的是vue框架，后端用的是nodejs加koa框架加websocket协议、http协议、mqtt协议，数据库有的就是mysql，现在让我说一下遇到的一些问题以及怎么解决的吧。
首先这个项目既然是为了物联网大赛，肯定是要与物理层进行通信的，大体就是物理层传数据，应用层接受数据并传控制底层设备的指令，很普遍，但有意思的是，最开始我们都是使用mqtt协议进行传输的，很简单，就调几个api就行，但因为25年物联网大赛的更新，要求我们跟上时代的步伐，项目要与大模型进行集成，就是说服务器要接入大模型，用于识别底层传的视频，并要求我们用http协议，好了，现在讲遇到的一些问题：
1.心跳问题：
    心跳反应是很常见的判断设备是否在线的方法，具体就是物理层要在特定时间向应用层发送一个心跳消息，那么应用层要设置一个定时器用来循环心跳检测，在定时器内又设置在几秒内如果未收到心跳消息则会判断设备掉线，然后我们需到的问题就是即便物理层一直在发送心跳，但应用层有时还是会判断设备掉线，这是因为在定时器时间间隔，如定时器是三秒一次，而判断的时间是2秒，那么就会存在一秒的间隔，这时候如果物理层的心跳也是在1秒以上的时间发送心跳，就会卡到这个间隔上，导致设备掉线，所以物理层就要把心跳设在1秒内发送

2.事件监听方法
    在项目要求实时更新数据，其实是很简单的事情就是通过ws来实现，但怎么将ws和http协议接受数据联系起来就是问题了，这时就要用到全局事件监听了，在nodejs中，我们可以通过events模块来实现，在events模块中，有一个EventEmitter类，这个类提供了一些方法，我们可以通过这个类来创建一个事件监听器，然后通过这个监听器来监听事件，当事件发生时，可以执行相应的回调函数。比如
    events.emit('update',{
        type:'sensor'
    })这是暴露一个名为update的事件，然后在ws中就要监听这个事件，当出现事件的时候，就提醒前端进行数据更新
     events.on('update',(payload)=>{
        let type = payload.type
        if(type == 'error'){
            ws.send(JSON.stringify('error'))
        }else if(type == 'sensor'){
            ws.send(JSON.stringify('sensor'))
        }else{
            ws.send(JSON.stringify('behavior'))
        }
    })

3.如何将不同的指令动态的渲染到页面上
    这真是当初难死我的地方，首先我们要根据指令的类型写好组件，然后在父组件上根据后端发过来的数据中的指令类型来动态渲染组件。
    getComponentName(direct) {
        switch (direct.f_type) {
        case '1':
            return 'SwitchComponent';
        case '2':
            return 'InputControl';
        case '3':
            return 'SliderComponent';
        case '4':
            return 'DateControl';
        case '5':
            return 'RadioControl';
        case '6':
            return 'DateRange';
        default:
            return 'div'; // 默认渲染一个 div
        }
    }
    <div class="content">
        <component v-for="direct in all_direct" 
        :key="direct.order" 
        :is="getComponentName(direct)" 
        :direct="direct" :d_no="null" @sendDirect="sendDirect"
        :style="'margin-left:' + 20 * direct.level + 'px;margin-top:10px;'"></component>
    </div>
    值得注意的是，在vue中如果key相同的话，那么组件会复用，不会重新渲染。所以在处理不同设备相同指令的时候，key值要取一个唯一值，不然指令不会刷新。然后还有在浏览器中ajax或是axios请求的次数是有限的，因为每次改变一个指令都要重新请求，请求次数过多就不再刷新了，所以我这里用的是ws

4.如何将表格中的地址改成能点击的超链接
    这里用到的是插槽
    <el-table-column v-for="(p,index) in header1" 
        :prop="p.prop" 
        :key="index"
        :label="p.label"
        width="180"
    >
        <template #default="scope">//名字为default，scope为上下文
            <a 
            v-if="p.label === '视频地址 '" 
            :href="scope.row[p.prop]" 
            target="_blank"  // 允许打开新窗口
            rel="noopener noreferrer"
            >
                点击播放
            </a>
            <span v-else>{{ scope.row[p.prop] }}</span>
        </template>
    </el-table-column>

koa2的学习

koa2的学习

今天我们来学习koa2框架，我愿称它为进阶版的express,但它比起express又轻量很多

首先我们要使用koa2肯定是要安装的，通过 npm i koa 进行安装

创建一个简单的服务器

然后和express一样的，我们先启动一个简单的http服务器，创建一个文件夹，在终端输入 npm init -y 进行初始化，生成package.json文件，再创建一个index.js文件，开始写服务器

import koa from ‘koa’

const app = new koa()

app.use(async ctx => {

​ ctx.body = ‘hello koa’

})

app.listen(3000,()=>{

​ console.log(‘服务器运行在 http://localhost:3000‘)

})

是不是很熟悉，这就是express中的样式啊，只是在express中的req，res换成了context上下文，更加方便，需要注意的是node默认是用commonJS语法的，但我这用了ES6的语法，是在packae.json文件中加了”type”:”module”这一句话

koa的中间件

koa中间件其实上面已经用到了，就是app.use()，是用来处理http请求与响应的函数

app.use(async (ctx,next) => {

​ console.log(‘中间件1’)

​ next()

​ console.log(‘中间件1 结束’)

})

app.use(async (ctx,next) => {

​ console.log(‘中间件2’)

​ next()

​ console.log(‘中间件2 结束’)

})

app.use(async (ctx,next) => {

​ console.log(‘中间件3’)

})

在这里我用了三个中间件，它的执行顺序是从上到下，当遇到next()函数的时候会停止当前中间件，执行下一个中间件，然后再倒着执行，也就是执行顺序是1 2 3 2 1

路由

又是路由，当涉及到网址相关部分的时候总是会避不开这个东西，接下来我们来学学koa中的路由，首先要用路由，先安装 npm i @koa/router

import koa from ‘koa’

import Router from ‘@koa/router’

const app = new koa()

const router = new Router()

//普通的get请求

router.get(‘/‘,async ctx => {

​ ctx.body = ‘Hello, Koa!’

})

//带请求参数的get请求 如http://localhost:3000/user?id=1&&name=zhangsan

router.get(‘/user’,async ctx => {

​ let {id,name} = ctx.query

​ ctx.body = User ID: ${id}, Name: ${name}

})

//带路径参数的get请求 如http://localhost:3000/user/2/wangwu

router.get(‘/user/:id/:name’,async ctx => {

​ let {id,name} = ctx.params

​ ctx.body = User ID: ${id}, Name: ${name}

})

//重定向

router.redirect(‘/test’,’/‘)

//使用路由中间件

app.use(router.routes())

//路由分组

const userRouter = new Router({prefix:’/user’})

userRouter.get(‘/‘,async ctx => {

​ ctx.body = ‘User List’

})

userRouter.get(‘/add’,async ctx => {

​ ctx.body = ‘添加用户’

})

userRouter.get(‘/delete’,async ctx => {

​ ctx.body = ‘删除用户’

})

app.use(userRouter.routes())

app.listen(3000,()=>{

​ console.log(‘服务器运行在 http://localhost:3000‘)

})

解决post请求

前面都是解决一些get请求，但在平常我们肯定会遇到post请求，接下来我们就来解决一下，首先安装插件 npm i @koa/bodyparser，用来解析post请求中以json和url格式发送的数据

import bodyparser from ‘@koa/bodyparser’

app.use(bodyparser())

router.post(‘/user’,async ctx =>{

​ console.log(ctx.request.body) // 打印请求体

​ let {id,name} = ctx.request.body

​ ctx.body = User ID: ${id}, Name: ${name}

})

错误处理和设置跨域

根据中间件的执行顺序，我们可以设置一个错误处理中间件用于解决在后续中间件中遇到问题时该怎么做，随便把跨域问题也讲了，当你想从你的服务器网址访问另一个网址时，浏览器有时会拦截这次请求，这时就要在服务器中设置允许跨域

import cors from ‘@koa/cors’

app.use(cors())

app.use(async (ctx,next)=>{

​ try{

​ await next() // 调用下一个中间件

​ }catch(err){

​ ctx.status = 500

​ ctx.body = ‘Internal Server Error’

​ }

})

图片上传

安装： npm i @koa/multer

import Multer from ‘@koa/multer’

import path from ‘path’

//配置磁盘存储

const storage = Multer.diskStorage({

​ destination:(req,file,callback)=>{

​ callback(null,’./upload’) //指定上传文件的目录

​ },

​ filename:(req,file,callback)=>{

​ callback(null,Date.now() + path.extname(file.originalname)) //指定上传文件的名称

​ }

})

const multer = Multer({//实例化Multer

​ storage,

​ fileFilter:(req,file,callback)=>{

​ if(file.mimetype.startsWith(‘image/‘)){ //只允许上传图片文件

​ callback(null,true)

​ }else{

​ callback(new Error(‘Only images are allowed’))

​ }

​ }

})

router.post(‘/upload’,multer.single(‘file’),async ctx =>{

​ let file = ctx.request.file

​ if(file){

​ console.log(file) // 打印上传的文件信息

​ ctx.body = File uploaded successfully: ${file.filename}

​ }

})
