koa2的学习

koa2的学习

今天我们来学习koa2框架，我愿称它为进阶版的express,但它比起express又轻量很多

首先我们要使用koa2肯定是要安装的，通过 npm i koa 进行安装

创建一个简单的服务器

然后和express一样的，我们先启动一个简单的http服务器，创建一个文件夹，在终端输入 npm init -y 进行初始化，生成package.json文件，再创建一个index.js文件，开始写服务器

import koa from ‘koa’

const app = new koa()

app.use(async ctx => {

​ ctx.body = ‘hello koa’

})

app.listen(3000,()=>{

​ console.log(‘服务器运行在 http://localhost:3000‘)

})

是不是很熟悉，这就是express中的样式啊，只是在express中的req，res换成了context上下文，更加方便，需要注意的是node默认是用commonJS语法的，但我这用了ES6的语法，是在packae.json文件中加了”type”:”module”这一句话

koa的中间件

koa中间件其实上面已经用到了，就是app.use()，是用来处理http请求与响应的函数

app.use(async (ctx,next) => {

​ console.log(‘中间件1’)

​ next()

​ console.log(‘中间件1 结束’)

})

app.use(async (ctx,next) => {

​ console.log(‘中间件2’)

​ next()

​ console.log(‘中间件2 结束’)

})

app.use(async (ctx,next) => {

​ console.log(‘中间件3’)

})

在这里我用了三个中间件，它的执行顺序是从上到下，当遇到next()函数的时候会停止当前中间件，执行下一个中间件，然后再倒着执行，也就是执行顺序是1 2 3 2 1

路由

又是路由，当涉及到网址相关部分的时候总是会避不开这个东西，接下来我们来学学koa中的路由，首先要用路由，先安装 npm i @koa/router

import koa from ‘koa’

import Router from ‘@koa/router’

const app = new koa()

const router = new Router()

//普通的get请求

router.get(‘/‘,async ctx => {

​ ctx.body = ‘Hello, Koa!’

})

//带请求参数的get请求 如http://localhost:3000/user?id=1&&name=zhangsan

router.get(‘/user’,async ctx => {

​ let {id,name} = ctx.query

​ ctx.body = User ID: ${id}, Name: ${name}

})

//带路径参数的get请求 如http://localhost:3000/user/2/wangwu

router.get(‘/user/:id/:name’,async ctx => {

​ let {id,name} = ctx.params

​ ctx.body = User ID: ${id}, Name: ${name}

})

//重定向

router.redirect(‘/test’,’/‘)

//使用路由中间件

app.use(router.routes())

//路由分组

const userRouter = new Router({prefix:’/user’})

userRouter.get(‘/‘,async ctx => {

​ ctx.body = ‘User List’

})

userRouter.get(‘/add’,async ctx => {

​ ctx.body = ‘添加用户’

})

userRouter.get(‘/delete’,async ctx => {

​ ctx.body = ‘删除用户’

})

app.use(userRouter.routes())

app.listen(3000,()=>{

​ console.log(‘服务器运行在 http://localhost:3000‘)

})

解决post请求

前面都是解决一些get请求，但在平常我们肯定会遇到post请求，接下来我们就来解决一下，首先安装插件 npm i @koa/bodyparser，用来解析post请求中以json和url格式发送的数据

import bodyparser from ‘@koa/bodyparser’

app.use(bodyparser())

router.post(‘/user’,async ctx =>{

​ console.log(ctx.request.body) // 打印请求体

​ let {id,name} = ctx.request.body

​ ctx.body = User ID: ${id}, Name: ${name}

})

错误处理和设置跨域

根据中间件的执行顺序，我们可以设置一个错误处理中间件用于解决在后续中间件中遇到问题时该怎么做，随便把跨域问题也讲了，当你想从你的服务器网址访问另一个网址时，浏览器有时会拦截这次请求，这时就要在服务器中设置允许跨域

import cors from ‘@koa/cors’

app.use(cors())

app.use(async (ctx,next)=>{

​ try{

​ await next() // 调用下一个中间件

​ }catch(err){

​ ctx.status = 500

​ ctx.body = ‘Internal Server Error’

​ }

})

图片上传

安装： npm i @koa/multer

import Multer from ‘@koa/multer’

import path from ‘path’

//配置磁盘存储

const storage = Multer.diskStorage({

​ destination:(req,file,callback)=>{

​ callback(null,’./upload’) //指定上传文件的目录

​ },

​ filename:(req,file,callback)=>{

​ callback(null,Date.now() + path.extname(file.originalname)) //指定上传文件的名称

​ }

})

const multer = Multer({//实例化Multer

​ storage,

​ fileFilter:(req,file,callback)=>{

​ if(file.mimetype.startsWith(‘image/‘)){ //只允许上传图片文件

​ callback(null,true)

​ }else{

​ callback(new Error(‘Only images are allowed’))

​ }

​ }

})

router.post(‘/upload’,multer.single(‘file’),async ctx =>{

​ let file = ctx.request.file

​ if(file){

​ console.log(file) // 打印上传的文件信息

​ ctx.body = File uploaded successfully: ${file.filename}

​ }

})
