docker部署云服务器

node为例，在工作目录上创建文件Dockerfile
文本内容:

基础镜像（Node.js官方镜像，选与你本地开发一致的版本）

FROM node:20.19.2-alpine

创建工作目录

WORKDIR /app

复制package.json和package-lock.json

COPY package*.json ./

安装依赖

RUN npm install –production # 生产环境安装（忽略devDependencies）

复制项目代码到工作目录

COPY . .

暴露服务端口（与你Node.js代码中监听的端口一致，例如3000）

EXPOSE 4000

启动命令（根据你的入口文件调整，例如node app.js）

CMD [“node”, “app.js”]

命令：
docker pull node:20.19.2-alpine #先单独拉取node官方镜像，避免网络问题，导致构建不了
docker build -t my-node-app:v1 . # 构建镜像，标签为my-node-app:v1 记得带后面的点，表示构建当前文件夹

docker run -d -p 3000:3000 –name my-node-service my-node-app:v1 运行测试

docker save -o my-node-app.tar my-node-app:v1 #打包镜像为rar文件

格式：scp 本地镜像路径 服务器用户名@服务器IP:目标路径

scp my-node-app.tar root@你的服务器公网IP:/home/

ssh root@公网ip # 登录云服务器

docker load -i /home/my-node-app.tar

查看镜像是否加载成功

docker images # 应能看到my-node-app:v1

运行容器，映射端口（服务器端口:容器端口，确保服务器端口未被占用）

docker run -d –name my-node-server -p 80:3000 –restart always my-node-app:v1

docker stop my-node-server #停止服务

docker rm my-node-server #移除服务
